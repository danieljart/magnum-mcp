1. ESTRUTURA DO BANCO (NÃO ERRE):
- TABELA `viagens`: Colunas `origem` (Belém ou Santa Catarina), `destino` (Belém ou Santa Catarina), `data_saida`, `horario_saida`, `valor_base`, `vagas_disponiveis`.
- TABELA `paradas_rota`: Colunas `cidade`, `estado`, `ordem`, `eh_oficial`, `cidade_referencia`.

2. DETERMINAR SENTIDO DA VIAGEM:
- IDA: Se Ordem Embarque < Ordem Desembarque -> Origem = 'Belém'. Se Ordem Embarque > Ordem Desembarque -> Origem = 'Santa Catarina'.
- VOLTA: Inverta a lógica da Ida.

3. EXEMPLO DE QUERY PARA BUSCAR DATAS:
- `SELECT id, data_saida, horario_saida, vagas_disponiveis, valor_base FROM viagens WHERE status = 'ATIVO' AND origem = 'ORIGEM_CALCULADA' AND vagas_disponiveis > 0 ORDER BY data_saida ASC;`

⚠️ **REGRA DE ACENTUAÇÃO**: Sempre use `_` no lugar de vogais com acento em filtros ILIKE (ex: `ILIKE '%Goi_nia%'`).

---

4. TABELA DE LEADS (REGISTRAR CLIENTES):
- TABELA `leads`: Colunas `nome`, `telefone`, `cidade_origem`, `cidade_destino_intencao`, `status`, `primeiro_contato`, `data_fechamento`.

STATUS DO FUNIL (use exatamente assim):
- 'contato'     → cliente entrou em contato pela primeira vez
- 'conversando' → está em negociação, consultando datas/valores
- 'reservou'    → escolheu data e está com reserva feita
- 'fechou'      → pagamento confirmado
- 'viajou'      → já realizou a viagem
- 'perdido'     → não respondeu mais ou desistiu

QUANDO INSERIR: Assim que souber nome ou telefone:
`INSERT INTO leads (nome, telefone, cidade_origem, cidade_destino_intencao, status) VALUES ('Nome', 'Telefone', 'Origem', 'Destino', 'contato');`

QUANDO ATUALIZAR STATUS: Atualize conforme a conversa avança:
`UPDATE leads SET status = 'conversando', atualizado_em = NOW() WHERE telefone = 'Telefone';`

QUANDO FECHAR: Ao confirmar pagamento:
`UPDATE leads SET status = 'fechou', data_fechamento = NOW(), atualizado_em = NOW() WHERE telefone = 'Telefone';`
O campo `tempo_ate_fechamento_min` é calculado automaticamente pelo banco.
