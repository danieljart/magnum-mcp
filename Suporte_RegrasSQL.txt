1. ESTRUTURA OFICIAL DO BANCO (SCHEMA) - USE EXATAMENTE ESTES NOMES:

TABELA: viagens
- id: UUID (PK)
- destino: TEXT
- data_saida: DATE
- horario_saida: TIME
- vagas_disponiveis: INTEGER (Campo para tirar/devolver vagas)

TABELA: reservas
- id: UUID (PK)
- codigo_reserva: TEXT (Ex: MAG-1234) -> Use este campo para buscas.
- nome_cliente: TEXT
- telefone: TEXT
- quantidade: INTEGER
- valor_total: NUMERIC
- status: TEXT ('PENDENTE', 'CONFIRMADO', 'CANCELADO')

2. COMO EXECUTAR (MÉTODO OBRIGATÓRIO):
Você DEVE preencher os parâmetros da ferramenta `run_sql` assim:

- `sql`: A query SQL.

### REGRAS DE SQL - SUPORTE MAGNUM (MANDATÓRIO)

A) CONSULTA FULL (USAR SEMPRE QUE RECEBER CONTATO):
```sql
SELECT 
    r.codigo_reserva, 
    r.status, 
    r.nome_cliente, 
    r.telefone, 
    r.quantidade, 
    r.valor_total, 
    r.viagem_id,
    v.destino, 
    v.data_saida, 
    v.horario_saida 
FROM reservas r 
JOIN viagens v ON r.viagem_id = v.id 
WHERE r.codigo_reserva = 'VALOR_ID' OR r.telefone = 'TELEFONE_NUMEROS';
```

B) CONFIRMAÇÃO DE PAGAMENTO (TRIGGER DE VENDA):
Ao confirmar um pagamento, você DEVE executar estas duas operações em sequência:

1. Atualizar Status da Reserva para 'CONFIRMADO':
```sql
UPDATE reservas 
SET status = 'CONFIRMADO' 
WHERE codigo_reserva = 'VALOR_ID' 
RETURNING viagem_id, quantidade, nome_cliente;
```

2. Baixar Vagas na Viagem (Subtraia a quantidade):
```sql
UPDATE viagens 
SET vagas_disponiveis = vagas_disponiveis - [QUANTIDADE] 
WHERE id = '[VIAGEM_ID_RETORNADO]';
```

C) CANCELAMENTO E ESTORNO DE VAGAS:
Se o cliente desistir ou houver erro no pagamento:
1. Mudar status para 'CANCELADO'.
2. Devolver as vagas na tabela `viagens` (vagas_disponiveis = vagas_disponiveis + [QUANTIDADE]).
